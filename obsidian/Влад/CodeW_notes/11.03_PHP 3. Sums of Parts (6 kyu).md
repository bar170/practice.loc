Let us consider this example (array written in general format):

`ls = [0, 1, 3, 6, 10]`

Its following parts:

```
ls = [0, 1, 3, 6, 10]
ls = [1, 3, 6, 10]
ls = [3, 6, 10]
ls = [6, 10]
ls = [10]
ls = []
```

The corresponding sums are (put together in a list): `[20, 20, 19, 16, 10, 0]`

The function `parts_sums` (or its variants in other languages) will take as parameter a list `ls` and return a list of the sums of its parts as defined above.

#### Notes

- Take a look at performance: some lists have thousands of elements.

Solution:

```php
function partsSums($ls) {
  
  $result[0] = 0;
  $count = count($ls);
  $sum = array_sum($ls);
  $result[0] = $sum;
  
  for ($i = 1; $i <= $count; $i ++) {
    $sum -= $ls[$i - 1];
    $result[$i] = $sum;
  } 
  
  return $result;
}
```

Сводка:

Функция получает массив и должна вернуть массив на один элемент больше. Каждый элемент возвращаемого массива должен содержать число равное сумме всех элементов первого массива, начиная от текущего положения. Пример: функция получает массив [4, 8, 16] и должна вернуть массив [28, 24, 16, 0]. Таким образом, возвращаемый массив всегда начинается с суммы всех его элементов и заканчивается [0]. 

В задаче есть подвох: в тестах дают невероятно огромный массив с большими значениями. Изначальное решение через array_sum на каждой итерации приводит к таймауту. Следовательно, нужно было написать алгоритм, который работает быстро. Для это вместо вычисления суммы всего массива на каждой итерации я вычитал из суммы всех элементов значение текущего элемента и перезаписывал значение суммы на новое.
