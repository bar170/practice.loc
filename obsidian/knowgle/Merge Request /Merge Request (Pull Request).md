# Merge Request (Pull Request)

```php
# Это не функция, а инструмент Git-платформ (GitHub, GitLab)
```

Merge Request (в GitLab) / Pull Request (в GitHub) — это инструмент для управления кодом, который позволяет разработчику запросить слияние изменений из одной ветки (например, feature-branch) в другую (например, master). Это не просто команда Git, а целый процесс, включающий обсуждение, ревью кода и автоматические проверки .

Функция принимает аргументом ветку с изменениями и ветку, куда нужно влить код. Возвращает согласованное и проверенное изменение в основной ветке.
В чем отличие от прямого пуша в мастер?

При прямом пуше (git push origin master) изменения попадают в основную ветку мгновенно и без контроля. При работе в команде это может привести к поломке общего кода .

Merge Request требует обязательного процесса:

    Код проверяют другие разработчики (code review) .

    Автоматически запускаются тесты (CI/CD), чтобы убедиться, что новый код ничего не сломал .

    Обсуждение изменений ведется в одном месте.

    Ветка master остается защищенной — напрямую пушить в нее часто запрещено настройками (Protected Branches) .

Пример использования (GitLab Flow)
php

# 1. Создаем новую ветку и работаем в ней
git checkout -b feature/new-header
git add .
git commit -m "Добавил новый хедер"
git push origin feature/new-header

# 2. Идем на сайт GitLab и создаем Merge Request из ветки feature/new-header в master

# 3. Коллеги смотрят код, оставляют комментарии.
#    Если нужно что-то поправить — делаем новые коммиты в эту же ветку и снова пушим.
#    Они автоматически добавятся в открытый Merge Request.

# 4. После одобрения и успешных тестов нажимаем кнопку "Merge".
#    Изменения вливаются в master, ветка feature/new-header обычно удаляется.

# Результат: Код попал в master, но прошел ревью и тесты.
# $result == "Код в master, команда довольна, продакшн работает"
