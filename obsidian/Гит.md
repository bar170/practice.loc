
### Как работаем

1) Каждую задачу начинаем с того что вытягиваем актуальную версию репозитория
2) От нее создаем новую ветку по типу (git checkout -b egor/задача_одним_словом)
3) Выполнили работу - закоммитили изменения в вашей ветке
4) пушим ветку
5) снова актуализируем ветку мастер на своих локалках
6) делаем черри-пик на актуальный мастер ваших коммитов, которые выполнили за сегоодня
7) пушим мастер

Пример:

1) Каждую задачу начинаем с того что вытягиваем актуальную версию репозитория


```
git fetch origin master
git checkout -B master origin/master
```


2) От нее создаем новую ветку по типу (git checkout -b egor/задача_одним_словом)


```
git checkout -b egor/zadanie_1
```


3) Выполнили работу - закоммитили изменения в вашей ветке


```
git add  файл1
git add  файл2
git add  файл3

```

Или же в шторме правой кнопкой по файлу кликаете и нажимаете git add (я так обычно делаю)

Коммитим


```
git commit -m 'Выполнил задание 1, заполнил отчет'
```


4) пушим ветку


```
git push origin egor/zadanie_1
```


Идите на гитхаб. Убедитесь что ветка ушла, что она появилась в списке веток

5) снова актуализируем ветку мастер на своих локалках

За это время (пока выполнял работу) мастер мог измениться, если кто-то зашлет туда свои коммиты, поэтому опять актуализируем


```
git fetch origin master
git checkout -B master origin/master
```


6) делаем черри-пик на актуальный мастер ваших коммитов, которые выполнили за сегоодня

В шторме (находясь на мастере) находите свою ветку, на каждом коммите, пкм
![[Pasted image 20260309204218.png]]

Делаете черри пик. Коммиты должны улететь на мастер

7) пушим мастер

Убедитесь, что коммиты попали на мастер. После чего пушим мастер.

Пока вы делали черри пик, может быть так что кто-то заслал на сервер более актуальны масте. Тогда снова актуализируетесь и черри пиками свои коммиты перекидываете на местер





Проверяем, что ваши изменения дошли. Визуально на гитхабе проверяйте, что появились изменения
